# Как включить порт 22 (SSH) в ISPManager

## Способ 1: Через раздел "Безопасность" / "Security"

1. **В левом меню найдите раздел:**
   - "Безопасность" (Security)
   - или "Файрвол" (Firewall)
   - или "SSH доступ" (SSH Access)

2. **Если есть раздел "SSH доступ":**
   - Откройте его
   - Найдите опцию "Разрешить SSH доступ" или "Enable SSH"
   - Включите переключатель
   - Сохраните изменения

3. **Если есть раздел "Файрвол" (Firewall):**
   - Откройте "Файрвол"
   - Найдите правило для порта 22
   - Если правила нет, создайте новое:
     - Порт: 22
     - Протокол: TCP
     - Действие: Разрешить (Allow)
   - Сохраните изменения

## Способ 2: Через настройки пользователя

1. **Перейдите в раздел:**
   - "Пользователи" (Users) в левом меню

2. **Выберите пользователя** (обычно root или ваш пользователь)

3. **Найдите вкладку:**
   - "SSH доступ" или "Shell доступ"
   - или "Дополнительно" (Advanced)

4. **Включите опцию:**
   - "Разрешить SSH доступ"
   - "Enable SSH"
   - или "Shell access"

5. **Сохраните изменения**

## Способ 3: Через терминал (если есть доступ)

Если у вас есть доступ к серверу другим способом, можно включить SSH через командную строку:

```bash
# Проверьте статус SSH
systemctl status sshd
# или
systemctl status ssh

# Включите SSH сервис
systemctl enable sshd
systemctl start sshd

# Проверьте, что порт 22 открыт
netstat -tuln | grep 22
# или
ss -tuln | grep 22
```

## Способ 4: Через файрвол (если порт заблокирован)

Если SSH сервис включен, но порт заблокирован файрволом:

1. **В ISPManager найдите:**
   - "Файрвол" (Firewall)
   - или "Безопасность" -> "Правила файрвола"

2. **Добавьте правило:**
   - Направление: Входящее (Incoming)
   - Протокол: TCP
   - Порт: 22
   - Действие: Разрешить (Allow)
   - Источник: Любой (Any) или конкретный IP

3. **Сохраните и примените правила**

## Проверка после включения

После включения SSH проверьте доступность:

```bash
# С вашего компьютера
ssh root@95.163.228.70
# или
ssh user@95.163.228.70
```

## Важные замечания:

- **Безопасность:** После включения SSH обязательно:
  - Используйте сильные пароли
  - Рассмотрите возможность использования SSH ключей вместо паролей
  - Ограничьте доступ по IP, если возможно
  - Отключите вход root по паролю (если возможно)

- **Если порт 22 не работает:**
  - Проверьте, установлен ли SSH сервер (обычно openssh-server)
  - Проверьте логи: `tail -f /var/log/auth.log` или `/var/log/secure`
  - Убедитесь, что файрвол не блокирует порт

## Альтернативные порты

Если порт 22 заблокирован провайдером, можно изменить порт SSH:
1. Отредактируйте `/etc/ssh/sshd_config`
2. Измените `Port 22` на другой порт (например, 2222)
3. Перезапустите SSH: `systemctl restart sshd`
4. Откройте новый порт в файрволе
