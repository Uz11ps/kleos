# Решения для доступа к серверу без SSH

## ✅ Сервер работает нормально!
- HTTP/HTTPS доступен
- API работает
- Проблема только в SSH (порт 22)

## Решение 1: Обратиться в техподдержку reg.ru (РЕКОМЕНДУЕТСЯ)

**Это самый быстрый способ:**

1. Откройте тикет в техподдержке reg.ru
2. Попросите:
   - Открыть порт 22 для SSH доступа
   - Или перезапустить PM2 процесс `kleos-api` на сервере
   - Проверить, почему фоновые задания ISPManager падают

**Преимущества:**
- Быстро (обычно отвечают в течение часа)
- Бесплатно
- Они имеют прямой доступ к серверу

## Решение 2: Использовать веб-консоль ISPManager

Если в ISPManager есть встроенная консоль:

1. Найдите раздел **"Консоль"** или **"Terminal"** в ISPManager
2. Через веб-консоль выполните:
   ```bash
   pm2 restart kleos-api
   pm2 status
   pm2 logs kleos-api --lines 50
   ```

## Решение 3: Управление через файловый менеджер ISPManager

Если есть файловый менеджер в ISPManager:

1. Найдите файлы проекта (обычно `/opt/kleos/kleos/server` или `/var/www/...`)
2. Проверьте логи PM2 через файловый менеджер
3. Можно редактировать файлы напрямую через веб-интерфейс

## Решение 4: Использовать другой порт для SSH

Если техподдержка может помочь:

1. Попросите их изменить SSH порт с 22 на другой (например, 2222)
2. Настроить файрвол для нового порта
3. Подключаться: `ssh -p 2222 root@95.163.228.70`

## Решение 5: Пересоздание сервера (НЕ РЕКОМЕНДУЕТСЯ)

**Пересоздание сервера поможет только если:**
- Сервер действительно сломан (но он работает!)
- Нужно полностью переустановить систему
- Есть проблемы на уровне ОС

**Недостатки:**
- Потеря всех данных (если нет бэкапа)
- Нужно заново настраивать все сервисы
- Долго (может занять несколько часов)
- Дорого (если платный сервер)
- Может не решить проблему с ISPManager

## Текущее состояние сервера

✅ **Работает:**
- HTTP/HTTPS (порты 80, 443)
- API endpoints (порт 3000)
- Health check: `https://api.kleos-study.ru/health` → `{"ok":true}`

❌ **Не работает:**
- SSH (порт 22) - закрыт файрволом
- ISPManager фоновые задания - падают с ошибкой

## Рекомендация

**НЕ пересоздавайте сервер!** Вместо этого:

1. **Сначала:** Обратитесь в техподдержку reg.ru - это самый быстрый способ
2. **Параллельно:** Проверьте, есть ли веб-консоль в ISPManager
3. **Если ничего не помогает:** Рассмотрите пересоздание только как последний вариант

## Команды для техподдержки

Попросите их выполнить:

```bash
# Перезапустить PM2
pm2 restart kleos-api

# Проверить статус
pm2 status

# Посмотреть логи
pm2 logs kleos-api --lines 50

# Открыть порт 22 в файрволе
# (зависит от используемого файрвола - iptables, firewalld, ufw)
```
