## 📦 Как собрать Release версию

### Шаг 1: Проверьте настройки
```bash
# Убедитесь, что версия обновлена в app/build.gradle.kts
versionCode = 7  # Увеличьте при следующем релизе
versionName = "1.0.6"  # Обновите версию
```

### Шаг 2: Соберите Release AAB
В Android Studio:
1. **Build** → **Generate Signed Bundle / APK**
2. Выберите **Android App Bundle**
3. Выберите ваш keystore файл
4. Выберите **release** build variant
5. Нажмите **Finish**

Или через командную строку:
```bash
./gradlew bundleRelease
```

Файл будет в: `app/build/outputs/bundle/release/app-release.aab`

### Шаг 3: Проверьте размер
После сборки проверьте размер файла. С ProGuard и shrinkResources размер должен быть значительно меньше.

## ⚠️ Важные замечания

### ProGuard может сломать код
Если после включения ProGuard приложение не работает:
1. Проверьте логи: `adb logcat | grep -i "proguard\|r8"`
2. Добавьте правила в `proguard-rules.pro` для проблемных классов
3. Используйте `-keep` для классов, которые не должны обфусцироваться

### Тестирование Release сборки
**ВАЖНО:** Обязательно протестируйте Release сборку перед публикацией:
```bash
# Установите Release APK на устройство
adb install app/build/outputs/apk/release/app-release.apk

# Или соберите APK для тестирования
./gradlew assembleRelease
```

### Проверка безопасности
- ✅ Токены не сохраняются в backup
- ✅ Только HTTPS соединения
- ✅ Нет hardcoded секретов в коде
- ✅ Логирование отключено в release
