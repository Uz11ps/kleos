# Инструкция по открытию порта 22 в ISPManager

## Проблема
SSH подключение зависает или выдает "Connection timed out", потому что порт 22 закрыт файрволом.

## Решение: Настройка файрвола в ISPManager

### Шаг 1: Откройте раздел файрвола
1. В ISPManager найдите раздел **"Управление сервером"** (Server management)
2. Нажмите на **"Брандмауэр"** (Firewall)

### Шаг 2: Создайте новое правило
1. Нажмите кнопку **"Добавить правило"** или **"Создать"** (Add rule / Create)
2. Заполните поля **ТОЧНО** так:

   **Протокол:** `TCP`
   
   **Порт:** `22`
   
   **Источник (Source Address):** `0.0.0.0/0` 
   ⚠️ **ВАЖНО:** Используйте именно `0.0.0.0/0` (ноль точка ноль точка ноль точка ноль слэш ноль), это означает "любой IP адрес"
   
   **Действие:** `Разрешить` (Allow)
   
   **Описание (необязательно):** `SSH access`

### Шаг 3: Сохраните правило
1. Нажмите **"Сохранить"** или **"Применить"** (Save / Apply)
2. **Дождитесь завершения фонового задания** - не закрывайте страницу, пока не увидите сообщение об успешном выполнении
3. Если появится ошибка "Longtask process was killed or died":
   - Попробуйте еще раз
   - Проверьте, что все поля заполнены правильно
   - Убедитесь, что у вас есть права администратора

### Шаг 4: Проверка
После сохранения подождите 30-60 секунд и попробуйте подключиться:
```powershell
ssh root@95.163.228.70
```

## Альтернативные способы доступа

Если порт 22 не открывается через ISPManager, попробуйте:

### Вариант 1: Веб-консоль ISPManager
- В ISPManager может быть встроенная веб-консоль (терминал)
- Найдите раздел "Консоль" или "Terminal" в ISPManager
- Через веб-консоль можно выполнить команды для перезапуска PM2

### Вариант 2: Другие порты
- Проверьте, может быть SSH работает на другом порту (например, 2222)
- Спросите у хостинг-провайдера о доступе к серверу

### Вариант 3: Управление через ISPManager
- Если есть возможность управлять процессами через веб-интерфейс ISPManager
- Найдите раздел управления процессами или сервисами

## Команды для перезапуска бэкенда (после успешного SSH подключения)

```bash
# Найти путь к серверу
find /opt /var/www /home -name 'package.json' -path '*/server/*' -type f 2>/dev/null | head -1 | xargs dirname 2>/dev/null || echo '/opt/kleos/kleos/server'

# Перезапустить PM2
pm2 restart kleos-api

# Проверить статус
pm2 status

# Посмотреть логи
pm2 logs kleos-api --lines 50

# Проверить health endpoint
curl http://localhost:8080/health
```

## Если ничего не помогает

1. Обратитесь в техподдержку хостинг-провайдера (reg.ru)
2. Попросите их открыть порт 22 для вашего IP адреса
3. Или попросите их перезапустить PM2 процесс `kleos-api` на сервере
